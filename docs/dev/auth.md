# 用户注册和登录功能

## 注册功能

### 需求分析

- 使用邮箱作为注册入口
- 引入图形验证码防止滥用邮件发送服务

### 前后端交互流程

1. 用户打开注册弹窗，前端展示图形验证码
2. 用户输入验证码和邮箱，前端将数据发送给后端，后端校验通过后向指定邮箱发送 6 位数字的邮箱验证码
3. 用户填写邮箱验证码和自定义密码，前端将邮箱、邮箱验证码、密码发送给后端，后端校验通过后保存用户信息，完成注册

### 涉及接口

- 获取图形验证码 - `GET /api/auth/captcha`
- 发送邮箱验证码 - `POST /api/auth/send-email-code`
- 用户注册 - `POST /api/auth/register`

## 登录功能

### 需求分析

- 使用邮箱和密码进行登录
- 引入图形验证码防止机器人、脚本

### 前后端交互流程

1. 用户打开登录弹窗，前端展示图形验证码
2. 用户输入邮箱、密码和图形验证码，前端将数据发送给后端，后端校验通过后生成对应的 JWT
3. 后端返回 JWT，前端将其保存，之后每次请求都携带 JWT

### 涉及接口

- 获取图形验证码 - `GET /api/auth/captcha`
- 用户登录 - `POST /api/auth/login`


## 获取图形验证码接口

### 接口定义

`GET /api/auth/captcha`

### 请求参数

无

### 响应结果

响应示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "captchaId": "a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8",
    "imageData": "data:image/png;base64,iVBORw0KGgo..."
  }
}
```

**响应参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| captchaId | string | 验证码唯一标识，用于后续提交时校验 |
| imageData | string | base64 编码的图片数据 |

**错误状态码说明：**

| code | message 示例 | 说明 |
|------|----------------------------------|--------------------------|
| 500  | "获取图形验证码失败，请稍后重试" | 获取图形验证码服务异常 |


## 发送邮箱验证码接口

### 接口定义

`POST /api/auth/send-email-code`

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱地址 |
| captchaId | string | 是 | 图形验证码 ID |
| captchaText | string | 是 | 图形验证码文本 |

**请求示例：**
```json
{
  "email": "user@example.com",
  "captchaId": "a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8",
  "captchaText": "A3b7"
}
```

### 响应结果

**响应示例：**
```json
{
  "code": 200,
  "message": "邮箱验证码已发送，请注意查收",
  "data": null
}
```

**错误状态码说明：**

| code | message 示例 | 说明 |
|------|----------------------------------|--------------------------|
| 40001 | "邮箱格式不正确" | 请求参数格式校验失败 |
| 40002 | "验证码错误，请重新输入" 或 "验证码已过期，请重新获取" | 图形验证码校验失败 |
| 50001 | "邮件发送失败，请稍后重试" | 邮件发送服务异常（如邮件配置有误） |


## 用户注册接口

### 接口定义

`POST /api/auth/register`

### 请求参数

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱地址 |
| emailCode | string | 是 | 邮箱收到的 6 位数字验证码 |
| password | string | 是 | 用户设置的密码，6-20 位（只能为字母、数字或 @ 符号） |

**请求示例：**
```json
{
  "email": "user@example.com",
  "emailCode": "654321",
  "password": "Password123@"
}
```

### 响应结果

**响应示例：**
```json
{
  "code": 200,
  "message": "账号注册成功",
  "data": null
}
```

**错误状态码说明：**

| code | message 示例 | 说明 |
|------|----------------------------------|--------------------------|
| 40001 | "邮箱格式不正确" 或 "邮箱验证码格式不正确" 等 | 请求参数格式校验失败 |
| 40003 | "邮箱验证码错误，请重新输入" 或 "邮箱验证码已过期，请重新获取" | 邮箱验证码校验失败 |
| 40004 | "该邮箱已被注册" | 存在相同邮箱的账号 |


## 用户登录接口

### 接口定义

`POST /api/auth/login`

### 请求参数

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱地址 |
| password | string | 是 | 用户密码，6-20 位（只能为字母、数字或 @ 符号） |
| captchaId | string | 是 | 图形验证码 ID |
| captchaText | string | 是 | 图形验证码文本 |

**请求示例：**
```json
{
  "email": "user@example.com",
  "password": "Password123@",
  "captchaId": "login_captcha_123456",
  "captchaText": "X8y9"
}
```

## 响应结果

**响应示例：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

**响应参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| accessToken | string | JWT 访问令牌，用于后续接口认证 |

**错误状态码说明：**

| code | message 示例 | 说明 |
|------|----------------------------------|--------------------------|
| 40001 | "邮箱格式不正确" 等 | 请求参数格式错误 |
| 40002 | "验证码错误，请重新输入" 或 "验证码已过期，请重新获取" | 图形验证码校验失败 |
| 40005 | "账号或密码错误" 或 "账号或密码错误，请检查邮箱是否正确" | 账号不存在或密码不匹配 |
| 40006 | "账号状态异常，暂时无法登陆" | 账号状态异常 |
